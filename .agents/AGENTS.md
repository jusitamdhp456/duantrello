# Rules

## Git Workflow
Whenever a coding task or milestone is completed, you MUST automatically:
1. Stage all changes (`git add .`).
2. Commit the changes with a descriptive message summarizing the work done.
3. Push the changes to the remote repository (`git push`).
If the push command hangs or fails (e.g., due to interactive credential prompts), inform the user so they can push manually.
